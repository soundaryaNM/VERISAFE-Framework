# CW_Test_Cov

Coverage analysis tool for executed C/C++ unit tests.

## What it does

- Runs coverage reporting using `gcovr` against an existing CMake build directory.
- Writes reports under the target repo (default: `tests/coverage_report/`).
- Uses the same base naming convention as test reports (default base: `interlocking_test_report`).

## Requirements

- Coverage data must exist in the build directory (e.g., `*.gcda` files). This requires building with coverage flags and running tests.
- Python + `gcovr` (installed via this tool's dependencies).

## Usage

From this workspace root:

- Run coverage:
  - `python -m ai_c_test_coverage.cli RailwaySignalSystem`

- Specify build/output dirs:
  - `python -m ai_c_test_coverage.cli RailwaySignalSystem --build-dir tests/build --output-dir tests/coverage_report`

- Override the base name:
  - `python -m ai_c_test_coverage.cli RailwaySignalSystem --report-base interlocking_test_report`

- HTML summary is generated by default. Disable it with:
  - `python -m ai_c_test_coverage.cli RailwaySignalSystem --no-html`

- Detailed per-file HTML is enabled by default (so you can click a file and see line-by-line coverage).
  Disable it with:
  - `python -m ai_c_test_coverage.cli RailwaySignalSystem --no-html-details`

- Include only specific files (repeatable). The report will only show these files:
  - `python -m ai_c_test_coverage.cli RailwaySignalSystem --include-file src/logic/Interlocking.cpp`

## Output files

- `tests/coverage_report/interlocking_test_report_coverage.txt`
- `tests/coverage_report/interlocking_test_report_coverage.xml`
- `tests/coverage_report/interlocking_test_report_coverage.html` (default)
